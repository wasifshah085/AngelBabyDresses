Project Overview

Angel Baby Dresses is a comprehensive e-commerce platform specializing in custom-made children's dresses. The platform enables customers to browse ready-made products, request custom designs, and manage orders with a unique split-payment system tailored for the Pakistani market.

50+ API Endpoints
15+ Database Models
30+ UI Components
2 Languages (EN/UR)

Key Business Features

Custom Design Orders

Customers can submit design ideas with reference images and receive personalized quotes from the admin team.

Split Payment System

50% advance payment via JazzCash/Easypaisa/Bank Transfer, remaining 50% + shipping as COD.

Age-Based Pricing

Dynamic pricing based on child's age range, with different prices for different size categories.

Bilingual Support

Full RTL support for Urdu language, making the platform accessible to local customers.

Target Audience

  • Primary: Parents looking for premium, custom-made dresses for their daughters
  • Secondary: Gift buyers for special occasions (birthdays, Eid, weddings)
  • Geographic Focus: Pakistan, with support for local payment methods

API Structure

Route Description
/api/auth/* User authentication, registration, password reset
/api/products/* Product CRUD, search, filtering, related products
/api/orders/* Order creation, status updates, payment processing
/api/cart/* Cart management for authenticated users
/api/reviews/* Product reviews with verified purchase check
/api/custom-designs/* Custom design submissions and quotes
/api/admin/* Admin-only operations and management

Security Measures

  • JWT Authentication: Secure token-based authentication with refresh tokens
  • Password Hashing: bcrypt with salt rounds for secure password storage
  • Input Validation: Express-validator for sanitizing all user inputs
  • Rate Limiting: Protection against brute force attacks
  • CORS Configuration: Restricted to allowed origins only

Database Design

The application uses MongoDB with Mongoose ODM for data modeling. The schema design supports complex relationships, embedded documents for performance, and indexing for fast queries.

Core Data Models

Technical Implementation Highlights

Performance Optimizations

React Query Caching

Intelligent data caching with stale-while-revalidate pattern. Reduces API calls by 60%+.

Image Optimization

Cloudinary transformations for responsive images. WebP format, lazy loading, blur placeholders.

Code Splitting

Route-based lazy loading with React.lazy() and Suspense for faster initial load.

Database Indexing

Compound indexes on frequently queried fields. Text indexes for search functionality.
